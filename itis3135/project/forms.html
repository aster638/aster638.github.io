Simple Forms

Understanding HTML Forms

HTML forms are essential for collecting user input on websites. They allow users to enter data that can be sent to a server for processing. In this module, we'll focus on the front-end structure and appearance of forms, not on how they process data.

Note: This module covers the HTML structure of forms only. For actual data processing and form submission, you would need server-side programming which is beyond the scope of this tutorial.

Basic Form Structure

Every HTML form starts with the <form> element. This element contains various input elements that collect different types of data:

HTML Code
Output
<form action="#" method="post">
    <!-- Form elements go here -->
    <input type="text" name="username">
    <input type="submit" value="Submit">
</form>

The <form> element has two important attributes:

  • action - Specifies where to send the form data when submitted
  • method - Specifies which HTTP method to use (usually "get" or "post")

Input Elements

The <input> element is the most versatile form control. Its behavior changes dramatically depending on the value of its type attribute.

Text Input

HTML Code
Output
<label for="username">Username:</label>
<input type="text" id="username" name="username" placeholder="Enter your username">

Password Input

HTML Code
Output
<label for="password">Password:</label>
<input type="password" id="password" name="password">

Email Input

HTML Code
Output
<label for="email">Email:</label>
<input type="email" id="email" name="email" required>

This field uses HTML5 validation to check for valid email format.

Checkboxes

HTML Code
Output
<div class="checkbox-group">
    <input type="checkbox" id="subscribe" name="subscribe" value="yes">
    <label for="subscribe">Subscribe to newsletter</label>
</div>
<div class="checkbox-group">
    <input type="checkbox" id="terms" name="terms" value="accepted" required>
    <label for="terms">I accept the terms and conditions</label>
</div>

Radio Buttons

HTML Code
Output
<div class="radio-group">
    <p>Select your preferred contact method:</p>
    <input type="radio" id="contact-email" name="contact" value="email" checked>

</div>
<div class="radio-group">
    <input type="radio" id="contact-phone" name="contact" value="phone">

Form Validation

HTML5 provides built-in form validation features through attributes like required, min, max, pattern, etc. These allow basic validation to occur without JavaScript.

Common validation attributes include:

  • required - Field must be filled out
  • min and max - Minimum and maximum values for number inputs
  • minlength and maxlength - Character limits for text inputs
  • pattern - Regular expression pattern the input must match
