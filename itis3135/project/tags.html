Basic HTML Tags

Headings

HTML provides six levels of headings, from <h1> (the most important) to <h6> (the least important). Headings help structure your content, making it easier to read and navigate.

HTML
Result
<h1>This is a Heading 1</h1>
<h2>This is a Heading 2</h2>
<h3>This is a Heading 3</h3>
<h4>This is a Heading 4</h4>
<h5>This is a Heading 5</h5>
<h6>This is a Heading 6</h6>

This is a Heading 1

This is a Heading 2

This is a Heading 3

This is a Heading 4

This is a Heading 5
This is a Heading 6

Best Practices for Headings

  • Use <h1> for the main title of the page (typically only one per page)
  • Structure your headings hierarchically (don't skip levels, like going from <h2> to <h4>)
  • Use headings to create a logical outline of your content
  • Headings are important for accessibility and SEO

Paragraphs

The <p> element defines a paragraph. Browsers automatically add margin space before and after each paragraph.

HTML
Result
<p>This is the first paragraph. HTML automatically adds space before and after this element.</p>
<p>This is the second paragraph. Notice how it starts on a new line with space above it.</p>

Links

The <a> (anchor) element creates hyperlinks to other web pages, files, locations within the same page, email addresses, or any other URL.

HTML
Result
<!-- Link to another website -->
<a href="https://www.example.com">Visit Example.com</a>

<!-- Link to another page in the same website -->
<a href="intro.html">Go to Introduction</a>

<!-- Link to a specific section on the same page -->
<a href="#images">Jump to Images section</a>

<!-- Link with target attribute to open in a new tab -->
<a href="https://www.example.com" target="_blank">Open in new tab</a>

Important Link Attributes

  • href: Specifies the URL or destination of the link
  • target: Specifies where to open the linked document (e.g., _blank opens in a new tab)
  • title: Provides additional information about the link (shown as a tooltip)
  • rel: Specifies the relationship between the current document and the linked document

Images

The <img> element embeds an image into the web page. It is an empty (self-closing) element, meaning it doesn't have a closing tag.

HTML
Result
<!-- Basic image -->
<img src="https://via.placeholder.com/300x200" alt="A placeholder image">

<!-- Image with width and height attributes -->
<img src="https://via.placeholder.com/150" alt="Small placeholder" width="150" height="150">

<!-- Image with a title (tooltip) -->
<img src="https://via.placeholder.com/200x100" alt="Wide placeholder" title="This is a tooltip">




Required Attributes for Images

  • src: Specifies the path to the image file
  • alt: Provides alternative text for the image (for accessibility and if the image cannot be displayed)

The alt attribute is crucial for accessibility, helping screen readers describe the image to visually impaired users. It's also displayed if the image fails to load.

Text Formatting

HTML provides several elements for formatting text to emphasize or highlight content.

HTML
Result
<!-- Strong importance -->
<p>This is <strong>very important</strong> text.</p>

<!-- Emphasized text -->
<p>This is <em>emphasized</em> text.</p>

<!-- Marked/highlighted text -->
<p>This is <mark>highlighted</mark> text.</p>

<!-- Subscript text -->
<p>H<sub>2</sub>O is the chemical formula for water.</p>

<!-- Superscript text -->
<p>2<sup>3</sup> equals 8.</p>

<!-- Code -->
<p>The <code>print()</code> function outputs text.</p>

This is very important text.

This is emphasized text.

This is highlighted text.

H2O is the chemical formula for water.

23 equals 8.

The print() function outputs text.

Semantic vs. Presentational Markup

It's important to use HTML elements for their semantic meaning, not just for their visual effect. For example:

  • Use <strong> for strong importance, not just to make text bold
  • Use <em> for emphasis, not just to make text italic

This distinction matters for accessibility and helps browsers, search engines, and other technologies understand your content better.
